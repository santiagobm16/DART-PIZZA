Interacción 1 - Crear pedido

Interacción 2 - Verificacion de productos

Interacción 3 - Proceso de pago

🪜 2. REST o Eventos

Interacción REST Evento ¿Por qué?
1 X La pizzería necesita esta interacción para saber si el pedido se genera o no
2 X La verificación de inventario puede llevarse a cabo sin una respuesta inmediata
3 X Se necesita de manera inmediata la confirmación del pago para continuar con el proceso

🪜 3. Síncrono o Asíncrono

Interacción Síncrono Asíncrono Justificación
1 X El sistema se detiene hasta que recibe la respuesta del servicio
2 X Este servicio puede seguir sin esperar una respuesta
3 X Se debe esperar la confirmacion del pago

🪜 4. Eventos Importantes del Sistema

1️⃣ Pedido

2️⃣ Pago

3️⃣ Invetario

🪜 5. Análisis de Fallos

Servicio de pagos
El cliente no podra completar la compra o el pedido se procesaria sin que se pagara.
Deberia continuar, pero no deberia dejar finalizar la compra.
- Se podria hacer que el sistema vuelva a intentarlo automaticamente.
- Se informaria al usuario y al administrador.
- Tambien el pago quedaria registrado como pendiente.

🪜 6. Service Discovery

Otros servicios no podrian encontrarlo y esto causaria fallos en la comunicacion.
A traves del servicio de deteccion de servicios.
Porque facilita que los servicio se encuentren entre si sin necesidad de direcciones IP fijas.
En pedidos, pagos, inventario y notificaciones.

🪜 7. Mini Diagrama

Dibujen:

Servicio A → Service Discovery → Servicio B

💡 Ayuda rápida
REST → petición directa
Evento → algo ocurrió
Síncrono → espera respuesta
Asíncrono → continúa sin esperar
Service Discovery → permite encontrar servicios automáticamente
🎤 El equipo debe poder explicar:
✔ una interacción importante
✔ un evento del sistema
✔ qué pasa si un servicio falla
✔ dónde usarían Service Discovery
